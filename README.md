## Project Overview

This system allows users to upload an MRI brain scan and automatically detect whether a tumour is present using a Convolutional Neural Network (CNN) trained on a labelled MRI dataset.

**Key capabilities:**
- Binary classification: *Tumour* vs *No Tumour*
- Confidence score and probability breakdown for each prediction
- CLAHE-enhanced MRI display for better visual inspection
- Fully CPU-compatible — no GPU required

## Setup & Installation

### Prerequisites

- Python 3.9 or later
- pip (comes with Python)
- ~2 GB free disk space (for PyTorch and dataset)

### Step 1 — Clone or download the project

```bash
git clone https://github.com/yourname/brain-tumor-detection.git
cd brain-tumor-detection
```

### Step 2 — Create a virtual environment (recommended)

```bash
# Windows
python -m venv venv
venv\Scripts\activate

# macOS / Linux
python3 -m venv venv
source venv/bin/activate
```

### Step 3 — Install dependencies

```bash
pip install -r requirements.txt
```

> **Note for M1/M2 Mac users:** PyTorch supports Apple Silicon natively.  
> The install command above works unchanged.

## Dataset Preparation

1. Download the **Brain MRI Images for Brain Tumor Detection** dataset from Kaggle:  
   👉 https://www.kaggle.com/datasets/navoneel/brain-mri-images-for-brain-tumor-detection

2. Extract and organise the images into:

```
dataset/
├── tumor/       ← paste all tumour images here   (~155 images)
└── no_tumor/    ← paste all healthy images here  (~98 images)
```

Supported image formats: `.jpg`, `.jpeg`, `.png`

## Training the Model

```bash

```

**Optional arguments:**

```bash
python train.py --epochs 30 --batch_size 16 --lr 0.001
```

| Argument | Default | Description |
|----------|---------|-------------|
| `--epochs` | 25 | Number of training epochs |
| `--batch_size` | 16 | Images per batch |

## Running the Web App

```bash
streamlit run app.py
```

Open your browser at: **http://localhost:8501**

**First launch without a trained model:**  
The app will automatically generate a demo (untrained) model so the UI is usable immediately. Run `train.py` afterwards to replace it with the real model.

## Deployment on Streamlit Cloud

### Step 1 — Push to GitHub

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/yourusername/brain-tumor-detection.git
git push -u origin main
```

> **Important:** Add your trained `model/brain_tumor_model.pt` to the repo  
> (or use Git LFS if the file exceeds 100 MB).

### Step 2 — Deploy on Streamlit Cloud

1. Go to [share.streamlit.io](https://share.streamlit.io)
2. Sign in with GitHub
3. Click **New app**
4. Select your repository, branch `main`, and main file `app.py`
5. Click **Deploy**

Streamlit Cloud will install `requirements.txt` automatically.

## Model Architecture

```
Input (3 × 224 × 224)
      │
ConvBlock 1  →  Conv2D(3→32)   + BN + ReLU + MaxPool → (32 × 112 × 112)
ConvBlock 2  →  Conv2D(32→64)  + BN + ReLU + MaxPool → (64 × 56 × 56)
ConvBlock 3  →  Conv2D(64→128) + BN + ReLU + MaxPool → (128 × 28 × 28)
ConvBlock 4  →  Conv2D(128→256)+ BN + ReLU + MaxPool → (256 × 14 × 14)
      │
Global Average Pooling → (256)
      │
FC(256→512) + ReLU + Dropout(0.4)
FC(512→256) + ReLU + Dropout(0.2)
FC(256→2)
      │
Softmax → [P(no_tumor), P(tumor)]
```

**Parameters:** ~1.6 million (fast to train on CPU)

**Alternative:** Switch to ResNet-18 transfer learning in `train.py` for higher accuracy on larger datasets.
